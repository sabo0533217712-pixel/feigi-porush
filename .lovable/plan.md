

# תוכנית שינויים במסך הגדרות האדמין

## סקירת השינויים

### 1. שעות עבודה לכל יום בנפרד + הפסקות מרובות

**DB Migration:**
- הוספת עמודה `day_schedules` (jsonb) ל-`business_settings` — מבנה:
```json
{
  "0": { "start": "09:00", "end": "18:00", "breaks": [{"start":"13:00","end":"14:00"}] },
  "1": { "start": "09:00", "end": "17:00", "breaks": [{"start":"12:00","end":"12:30"},{"start":"15:00","end":"15:15"}] }
}
```
- עמודות `start_time`, `end_time`, `break_start`, `break_end` הקיימות יישארו כ-fallback

**UI:** במקום כרטיס "שעות פעילות" גלובלי, כל יום עבודה מסומן (checkbox) יקבל שדות: שעת פתיחה, שעת סגירה, ורשימת הפסקות דינמית (כפתור "הוסף הפסקה" / מחיקה)

**עדכון לוגיקת הזמנה:** `ClientBooking.tsx` ו-`AdminCalendar.tsx` ישתמשו ב-`day_schedules` במקום בשעות הגלובליות

### 2. הסרת צבע ראשי/משני + חילוץ צבעים מהלוגו

- הסרת שדות `primary_color` ו-`secondary_color` מה-UI (הם יישארו ב-DB כ-fallback)
- הסרת שדה פונט מה-UI
- כשמעלים לוגו, המערכת תחלץ את הצבע הדומיננטי מהתמונה באמצעות canvas ותעדכן את CSS variables של האתר בהתאם (גוון ראשי, גוון משני, גרדיאנטים) — כך הלוגו "קובע" את פלטת הצבעים
- הצבעים המחולצים יישמרו ב-`business_settings` אוטומטית

### 3. תצוגת לוגו ותמונות גלריה עם מחיקה

**לוגו:** אחרי כפתור ההעלאה, יוצג preview של הלוגו הנוכחי (מ-Supabase Storage) עם כפתור מחיקה

**גלריה:** אחרי כפתור ההעלאה, יוצגו כל תמונות הגלריה בגריד קטן (thumbnails), כל תמונה עם כפתור X למחיקה (קריאה ל-`supabase.storage.from('gallery').remove([fileName])`)

## קבצים שישתנו

| קובץ | שינוי |
|---|---|
| Migration SQL | הוספת `day_schedules` jsonb ל-`business_settings` |
| `AdminSettings.tsx` | שכתוב מלא: ימים עם זמנים, הפסקות מרובות, הסרת צבעים, תצוגת לוגו/גלריה עם מחיקה, חילוץ צבעים מלוגו |
| `ClientBooking.tsx` | שימוש ב-`day_schedules` לחישוב זמינות לפי יום |
| `AdminCalendar.tsx` | שימוש ב-`day_schedules` לתצוגת שעות לפי יום |
